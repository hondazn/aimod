# Redundancy Check - 冗長性チェック

## 入力の取得

```text
$ARGUMENTS
```

入力がファイルパスの場合はファイルを読み込む。入力が直接テキストの場合はそのまま使用する。

## 評価プロセス

文章全体を通読した上で、以下の4つの観点でそれぞれ評価する。スコアは0.0〜5.0の0.5刻みで付ける。

### 観点1: 情報密度（Information Density）

1文あたりにどれだけ意味のある新しい情報が含まれているかを測る。

- **5.0**: すべての文が固有の情報を持ち、削除すると意味が欠落する
- **4.0**: ほぼすべての文が有用だが、わずかに薄い箇所がある
- **3.0**: 情報量のある文と薄い文が混在している
- **2.0**: 多くの文が既出情報の言い換えや自明な記述
- **1.0**: 大半の文が情報をほとんど追加していない

チェックポイント:
- 同じ情報を異なる表現で繰り返していないか
- 「〜ということです」「〜と言えるでしょう」のような情報を追加しない文末表現がないか
- 具体例や数値なく抽象論が続いていないか

### 観点2: 簡潔性（Conciseness）

個々の表現レベルで、より短く同じ意味を伝えられる箇所がないかを測る。

- **5.0**: すべての表現が必要最小限で、削れる言葉がない
- **4.0**: ほぼ簡潔だが、数箇所に冗長な修飾や回りくどい表現がある
- **3.0**: 冗長表現が目立つ箇所がいくつかある
- **2.0**: 多くの文に不要な修飾語・接続詞・前置きがある
- **1.0**: 全体的に回りくどく、半分以下の文字数で同じ内容を伝えられる

チェックポイント:
- 「〜することができる」→「〜できる」のような冗長パターン
- 不要な「という」「ような」「といった」の多用
- 「まず最初に」「一番最後に」のような重言
- 「非常に」「とても」「極めて」等の効果の薄い強調語
- 必要以上に長い前置きや断り書き

### 観点3: 論理効率（Logical Efficiency）

論理展開に無駄がないか、同じ主張を複数回していないかを測る。

- **5.0**: 論理が一直線に進み、各段落が固有の役割を果たしている
- **4.0**: ほぼ効率的だが、軽微な重複や遠回りがある
- **3.0**: 一部で論点が戻ったり、同じ主張が形を変えて繰り返されている
- **2.0**: 複数の段落が実質的に同じことを述べており、統合可能
- **1.0**: 循環論法や大幅な論理的重複がある

チェックポイント:
- 結論を先に述べた後、同じ結論に再度到達する冗長な論証がないか
- 「つまり」「要するに」の後に新しい情報がなく、単なる繰り返しになっていないか
- 異なるセクションで同じ論点を扱っていないか

### 観点4: 構造最適性（Structural Optimality）

見出し・セクション構成が内容に対して必要十分かを測る。

- **5.0**: 各セクションが明確な役割を持ち、過不足がない
- **4.0**: 構造はほぼ適切だが、統合や分割が有効な箇所が少しある
- **3.0**: いくつかのセクションが薄すぎる、または類似セクションの統合が望ましい
- **2.0**: 構造に明らかな非効率があり、再構成が必要
- **1.0**: セクション分けが内容と整合しておらず、根本的な再構成が必要

チェックポイント:
- 1〜2文しかないセクションが多くないか（他と統合すべき）
- 類似した内容の見出しが並んでいないか
- 深すぎるネストで読者の認知負荷が高くないか

## 出力フォーマット

以下のテンプレートに従って結果を出力する。

```
## 冗長性チェック結果

### スコアサマリー

| 観点 | スコア | 判定 |
|------|--------|------|
| 情報密度 | X.X / 5.0 | 🟢良好 / 🟡改善余地あり / 🔴要改善 |
| 簡潔性 | X.X / 5.0 | 🟢良好 / 🟡改善余地あり / 🔴要改善 |
| 論理効率 | X.X / 5.0 | 🟢良好 / 🟡改善余地あり / 🔴要改善 |
| 構造最適性 | X.X / 5.0 | 🟢良好 / 🟡改善余地あり / 🔴要改善 |
| **総合** | **X.X / 5.0** | |

### 主な問題箇所（スコアが低い観点から優先的に記述）

#### [観点名]: [問題の要約]

- **該当箇所**: 原文の引用（長い場合は冒頭と末尾を示す）
- **問題**: 何が冗長なのか具体的に説明
- **改善案**: 具体的な書き換え案を提示

（問題箇所ごとに繰り返す。重要度の高いものから最大10件）

### 総評

文章全体の冗長性について2〜3文で総括し、最も効果的な改善アクション（1〜3つ）を提案する。
```

判定基準:
- 4.0以上: 良好
- 2.5〜3.5: 改善余地あり
- 2.0以下: 要改善

総合スコアは4観点の平均値（小数第1位まで）とする。

## 注意事項

- 文章のジャンル（技術文書、ブログ、報告書等）を考慮してスコアリングする。技術文書では正確性のための繰り返しが許容される場合がある。
- 改善案は「削るだけ」でなく、情報を保ったまま圧縮する方向で提案する。意味が変わる削除は提案しない。
- 文体や個人の好みには踏み込まず、客観的に測定可能な冗長性のみを評価する。
- 短い文章（500文字未満）の場合、構造最適性の評価は省略してよい。
