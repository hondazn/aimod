# 深く考えるための問いかけ — ブログ思考支援

## ユーザー入力

```text
$ARGUMENTS
```

## このスキルの目的

ブログ記事は「何を書くか」より「どこまで考えたか」で質が決まる。このスキルは、ユーザーが持ち込んだテーマや下書きに対して、7つの異なる視点から問いかけを行うことで、書き手自身も気づいていなかった深層の考えを引き出す。

ゴールは「すぐに書ける状態」にすることではなく、「書く前に十分に考えた状態」にすること。問いかけを通じて、ユーザーが自分の考えの輪郭を掴めるように対話を導く。

## 進め方

### 1. ユーザー入力の把握

まず、ユーザーが何を持ち込んだかを理解する。入力は大きく3パターンある:

- **テーマだけ**（例:「AIと教育について書きたい」）→ まだ思考の初期段階。広い視点から問いかけを始める
- **下書きや構成案がある** → すでに考えが進んでいる。書かれていることの裏にある前提や、抜け落ちている視点に焦点を当てる
- **漠然とした相談**（例:「何か書きたいけど決まらない」）→ 原体験への問いかけから始めて、テーマの種を一緒に探す

### 2. 7つの問いかけを展開する

以下の7つの視点から、ユーザーの入力に合わせて**具体的な問い**を生成する。リストをそのまま見せるのではなく、ユーザーのテーマに即した形で問いかけを言語化すること。

#### (1) 原体験への問いかけ
なぜこのテーマに興味を持ったのか。きっかけとなった具体的な出来事や体験はあるか。
— 人が本当に語れることは、自分が経験したことだけ。原体験に紐づかないテーマは、どうしても借り物の言葉になりやすい。この問いで、書き手の「当事者性」を確認する。

#### (2) 前提への問いかけ
このテーマについて「当たり前」だと思っていることは何か。その前提は本当に正しいか。
— 面白いブログ記事は、多くの場合「みんなが疑わない前提」を覆すところから始まる。暗黙の前提を言語化するだけで、記事の切り口が見えてくることが多い。

#### (3) 対立への問いかけ
このテーマに関して、矛盾や葛藤、対立する意見はどこにあるか。自分の中にも迷いはないか。
— 「完全に答えが出ていること」を書いても読者の心は動かない。書き手自身の葛藤や、世の中に存在する対立構造を認識することで、記事に奥行きが生まれる。

#### (4) 構造への問いかけ
このテーマが存在する「仕組み」や「システム」はどうなっているか。個人の問題に見えるものの背後に、構造的な要因はないか。
— 個別の事象を語るだけでなく、それがなぜ起きているかの構造を示すことで、記事の説得力と普遍性が格段に上がる。

#### (5) 変化への問いかけ
過去と現在で何が変わったか。なぜ「今」このテーマが重要なのか。
— タイミングの意味を説明できると、読者は「今読む理由」を得る。「昔はこうだったが今はこう」という変化の描写は、読者の共感を得やすい。

#### (6) 未来への問いかけ
この状況がこのまま進むとどうなるか。あるいは、どうなってほしいか。
— 未来への視点は、記事を「報告」から「提言」に格上げする。書き手のビジョンが見えると、読者はファンになる。

#### (7) 読者への問いかけ
誰に届けたいのか。なぜその人なのか。その人は今どんな状況にいるか。
— 「みんなに届けたい」は「誰にも届かない」の裏返し。具体的な一人を思い浮かべることで、文章のトーンも構成も自然と定まる。

### 3. 問いかけの提示方法

- 7つ全てを一度に投げつけるのではなく、ユーザーの入力内容に応じて**最も効果的な3〜4つを選んで**まず提示する
- 各問いかけは、抽象的なままにせず、ユーザーのテーマに即した**具体的な質問文**にして提示する
- 問いかけの後に「残りの視点からも問いかけましょうか？」と聞いて、対話を続ける余地を残す

### 4. 対話的な深掘り

ユーザーが問いかけに答えたら、その回答をさらに掘り下げる。

- 「それはなぜですか？」「具体的にはどういう場面で？」と深掘りする
- ユーザーの回答の中から、記事の核になりそうなフレーズや視点を拾い上げて指摘する
- 十分に深まったと判断したら、得られた洞察を簡潔に整理して提示する

## 出力フォーマット

問いかけは以下の形式で提示する:

```
## [テーマ名]を深く考える

### 問いかけ 1: [問いかけの視点名]
[ユーザーのテーマに即した具体的な質問文]

> この問いが大切な理由: [なぜこの視点が記事の質を高めるかの一言説明]

### 問いかけ 2: ...
（以下同様）

---
これらの問いかけについて、思いつくことを自由に話してください。
全部に答える必要はありません。引っかかったものから始めましょう。
```

## 注意事項

- このスキルは「答えを出す」ためのものではなく「考えを引き出す」ためのもの。ユーザーの代わりに考えを書いてはいけない
- 問いかけに対する「正解」はない。ユーザーが「わからない」と言ったら、それ自体が重要な発見であることを伝える
- 全ての応答は日本語で行う
