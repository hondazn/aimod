## AI臭さの典型パターンと対処

### 冗長な導入

```
【検出パターン】
- 「〜について説明します」
- 「〜を見ていきましょう」
- 「それでは早速〜」
- 「まずはじめに〜」
- 「本記事では〜」

【なぜ問題か】
読者はすでにタイトルやリンクを見てこの文章に辿り着いている。
「何について書くか」を改めて宣言する必要はない。
本題が遅れるほど、読者は離脱する。

【対処】
削除して本題から始める。冒頭は具体的な事実、問い、またはエピソードで。
```

### 過剰な修飾

```
【検出パターン】
- 「非常に」「極めて」「大変」の多用
- 「〜と言っても過言ではありません」
- 「実に」「まさに」「本当に」の連発

【なぜ問題か】
強調語を重ねても説得力は上がらない。むしろ「根拠がないから言葉で盛っている」
という印象を与える。本当に重要なことは、事実で語れるはず。

【対処】
削除するか、具体的な根拠に置換。
悪い例: 「非常に効率的なツールです」
良い例: 「作業時間が半分になるツールです」
```

### 曖昧な断定回避

```
【検出パターン】
- 「〜かもしれません」の連発
- 「〜と思われます」の多用
- 「〜ではないでしょうか」の繰り返し

【なぜ問題か】
1回なら適切な留保だが、連発すると「自信がない」「責任を取りたくない」
という印象になる。読者は「結局どうなの？」と感じる。

【対処】
断言するか、条件を明示する。
悪い例: 「便利かもしれません」
良い例: 「初心者には便利だ」「慣れている人には不要だが、初めてなら使う価値がある」
```

### パターン化した構文

```
【検出パターン】
- 「〜という点で重要です」
- 「〜が大切です」「〜が必要です」の連続
- 「〜することが可能です」

【なぜ問題か】
同じ構文の繰り返しは、内容ではなくフォーマットで文章を埋めている証拠。
読者は3回目あたりで「またこのパターンか」と感じ、中身を読まなくなる。

【対処】
具体的な行動・結果に言い換える。
悪い例: 「コミュニケーションが重要です」
良い例: 「週1回の1on1で認識のズレが減った」
```

### 感情の押し付け

```
【検出パターン】
- 「素晴らしい」「驚くべき」「感動的な」
- 過度に肯定的な表現
- 読者の感情を代弁する表現（「誰もが興奮せずにはいられません」）

【なぜ問題か】
「すごい」と書かれても読者は「すごい」と感じない。
感情は事実を読んで読者自身が抱くもの。押し付けられると逆に冷める。

【対処】
事実を書き、判断は読者に委ねる。
悪い例: 「驚くべき成果を上げました」
良い例: 「前年比200%の成果が出た」
```

## 文体統一ルール

### です/ます調

丁寧で親しみやすい。ブログ、説明文、メール、カスタマー向け文書に適する。

```
文末: 「〜です」「〜ます」「〜でしょう」「〜しましょう」「〜ください」
```

### だ/である調

客観的で力強い。論文、報告書、技術文書、社内ドキュメントに適する。

```
文末: 「〜だ」「〜である」「〜となる」「〜といえる」「〜すべきだ」
```

### 混在を防ぐルール

1. 文書の最初の数文で使われている文体を基準とする
2. 引用文・会話文を除き、本文は一貫させる
3. 箇条書き内も本文と統一する
4. 見出しと本文で文体を変えない
5. 原文を校正する場合、原文の文体に合わせる
