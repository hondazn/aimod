# Natural Writing - 自然な日本語文章

人が書いた文章には「著者の声」がある。読み手は無意識にそれを感じ取り、信頼や共感につなげる。AI生成文にありがちな定型表現や過剰修飾は、この「声」を消してしまう。このスキルは、著者らしさが伝わる自然な日本語を生み出すためのガイドである。

## モード判定

ユーザーの入力を読み、以下の3つのうち最適なモードを選ぶ。明示的な指示がなくても、文脈から判断してよい。

| モード | いつ選ぶか | ゴール |
|--------|-----------|--------|
| **新規作成** | テーマ・要点が提示され「書いて」「作って」等の指示がある | 自然な文章をゼロから生成する |
| **校正** | 既存文章が提示され、改善を求められている（デフォルト） | 原文の意図を保ちつつ自然さを高める |
| **診断** | 「診断」「チェック」「分析」「評価」等の語を含み、文章を直さず評価だけ求められている | 問題点と良い点を指摘する（書き換えない） |

迷ったら校正モードを選ぶ。ユーザーが文章を貼り付けただけなら、たいてい「直してほしい」という意味である。

## 校正モード

校正の核心は「原文の意図を壊さずに、読み手に届く文章にする」こと。著者が伝えたかったことを汲み取り、それをより効果的に表現し直す。機械的な置換ではなく、文脈を理解した上で判断する。

### 思考の流れ

1. **全体を通読する** - 何を伝えたい文章か、主旨を把握する
2. **AI臭さを検出する** - 定型表現、過剰修飾、曖昧な断定回避をマークする
3. **文体の一貫性を確認する** - です/ます調とだ/である調の混在がないか
4. **具体化の余地を探す** - 抽象的な形容詞を、具体的な数値・事例・理由に置き換えられないか
5. **冗長表現を刈る** - 意味を変えずに削れる語句を削る
6. **声に出して読む感覚で確認する** - 不自然なリズムや言い回しがないか

### AI臭さの検出と対処

AI生成文が不自然に感じられるのは、「誰が書いても同じになる」表現が多いからである。以下のパターンを検出したら、著者の声が出る表現に書き換える。

| 検出パターン | なぜ問題か | 対処方針 |
|-------------|-----------|---------|
| 冗長な導入「〜について説明します」「本記事では〜」 | 読者は前置きを読み飛ばす。本題が遅れるだけ | 削除して本題から始める |
| 過剰修飾「非常に」「極めて」「まさに」の多用 | 具体性がないまま強調しても説得力がない | 具体的な理由・数値に置換するか、削除 |
| 曖昧な断定回避「〜かもしれません」「〜と思われます」の連発 | 責任逃れに見え、読者の信頼を損なう | 断言するか、条件を明示する（「初心者には〜」等） |
| パターン構文「〜という点で重要です」「〜が大切です」の繰り返し | テンプレート感が出る | 具体的な行動・結果に言い換える |
| 感情の押し付け「素晴らしい」「驚くべき」 | 読者が自分で判断する余地を奪う | 事実を書き、判断は読者に委ねる |

### 冗長表現の刈り込み

日本語は丁寧にしようとすると冗長になりやすい。意味が変わらないなら短い方がよい。

- 「〜することができます」→「〜できます」
- 「〜という形になります」→「〜です」
- 「基本的には」「一般的に」→ 削除するか、具体的な条件に置き換える
- 「〜を行う」→「〜する」
- 「〜についての」→「〜の」

一文の目安は60字以内。長くなったら分割を検討する。

簡潔にすべきでない場面:
- ビジネスメールで相手への配慮が必要なとき
- 複雑な概念を噛み砕いて説明しているとき
- 文章全体が短文ばかりでぶつ切り感があるとき

### 文体の統一

ひとつの文書内で文体が揺れると、読者は落ち着いて読めない。

- です/ます調とだ/である調を混ぜない（引用文は例外）
- 箇条書き内も本文と統一する
- 文体の選択基準: ブログ・メール・説明文 → です/ます調、報告書・論文・技術文書 → だ/である調
- 原文の文体が統一されているなら、それに合わせる

### 具体化

「重要」「便利」「効果的」は情報量ゼロの言葉である。読者が知りたいのは「なぜ重要か」「どう便利か」「何がどう変わるか」。

- 抽象的な形容 → 数値、事例、比較に置換
- 主観的な評価 → 客観的な事実に書き換える
- ただし、すべてを具体化する必要はない。文章のリズムや読みやすさとのバランスを取る

### 校正チェックリスト

校正時に上から順に確認する。すべての項目に該当する必要はない。

```
[ ] 文章全体の主旨を把握したか
[ ] AI臭い定型表現はないか（冗長な導入、過剰修飾、曖昧な断定回避、パターン構文、感情押し付け）
[ ] 冗長な修飾語・婉曲表現はないか
[ ] 文体は統一されているか（です/ます調 or だ/である調）
[ ] 一文が長すぎないか（目安60字。リズムを考慮）
[ ] 抽象的な形容を具体化できないか
[ ] 読者が行動・判断できる情報が含まれているか
[ ] 不要な導入句・締め句はないか
[ ] 原文の意図や著者の個性を損なっていないか
```

## 新規作成モード

ゼロから文章を書く場合、以下の手順で進める。

### 事前確認

ユーザーが明示していなければ、以下を確認するか文脈から推測する:
- **読者**: 誰に向けた文章か（専門家 / 一般読者 / 社内 / 顧客）
- **目的**: 何を伝えたいか（情報提供 / 説得 / 報告 / 依頼）
- **文体**: です/ます調 / だ/である調 / カジュアル
- **分量**: 大まかな長さ

読者層に応じた調整:
- 一般読者向け: 専門用語を避け、身近な例えを使う
- 専門家向け: 冗長な説明を省き、専門用語で簡潔に書く
- 社内向け: 組織の共通言語を使い、外部向けの丁寧さは不要

### 執筆の原則

1. **冒頭は具体的に** - 抽象的な前置きではなく、読者が「自分に関係ある」と感じる情報から始める
2. **一文一意** - 1つの文で1つのことだけ伝える。接続助詞でつなぎすぎない
3. **構成は伝えたい順** - 重要なことから書く。結論を後回しにしない
4. **自分の声で書く** - テンプレート表現を使わず、この文章でしか言えないことを書く
5. **結論は明確に** - 「〜ではないでしょうか」で終わらせない

### やってはいけないこと

- 「〜について書きます」「本記事では〜」で始める（読者にとって無意味）
- 「非常に」「極めて」を根拠なく使う
- 「いかがでしたでしょうか？」「最後までお読みいただきありがとうございます」等のテンプレート的な締め

## 診断モード

文章を書き換えず、品質を評価して問題点と改善方向を示す。

### 自然さスコア基準

| スコア | 状態 | 判断の目安 |
|--------|------|-----------|
| 5.0 | 自然 | AI臭さなし、具体的、文体統一、著者の声がある |
| 4.0 | ほぼ自然 | 軽微な問題が1-2箇所ある程度 |
| 3.0 | 改善余地あり | AI臭い表現や抽象的な形容が複数ある |
| 2.0 | 不自然 | 定型表現が目立ち、著者の声が感じられない |
| 1.0 | 機械的 | テンプレートの穴埋めのような文章 |

## 出力形式

モードごとに以下の形式で出力する。ただし、形式に縛られすぎず、ユーザーにとって最も読みやすい出力を優先する。

### 新規作成モード

```
[生成した文章]

---
**構成メモ**
- 意識した点を簡潔に
```

### 校正モード

```
[改善後の文章]

---
**主な変更点**
- 変更内容と理由を簡潔に

**残した箇所**（原文を意図的に残した場合）
- 残した理由
```

変更点では「何を変えたか」だけでなく「なぜ変えたか」を一言添える。原文の表現を意図的に残した場合、その判断理由も示す。これにより、ユーザーが自分の文章力を向上させるヒントになる。

### 診断モード

```
**自然さスコア: [1.0-5.0]**

**検出された問題**
1. [問題] - 該当箇所の引用 - 改善の方向性

**良い点**
- 評価できる点（必ず見つけて伝える）
```

良い点を必ず挙げる理由: 文章を提出してチェックを求めるユーザーは、自分の文章に不安を持っていることが多い。問題の指摘だけでは「全部ダメ」という印象を与えてしまう。良い点を明示することで、著者が自分のスタイルの強みを自覚し、それを伸ばせるようになる。

## 判断に迷ったとき

### 丁寧さ vs 簡潔さ

ビジネスメールや顧客対応では、冗長に見える表現が実は必要な丁寧さであることがある。

```
「ご確認いただけますと幸いです」
→ 簡潔にすると「確認してください」だが、ビジネス文脈では不適切なことが多い
→ 判断基準: 読者との関係性と文書の目的による
```

### 断言 vs 留保

すべてを断言すれば良いわけではない。不確実なことを断言するのは誠実でない。

```
判断基準:
- 事実 → 断言する
- 個人の意見 → 「〜だと考えている」（1回は問題ない）
- 不確実な情報 → 条件を明示する（「〜の場合は」）
- 推測が連発しているとき → 最も確度の高いものだけ残し、他は断言 or 削除
```

### 著者の個性 vs 読みやすさ

校正で著者の個性を消しすぎると、「正しいが無味乾燥な文章」になる。

```
判断基準:
- 独特だが意味が通る表現 → 残す
- 独特で意味が伝わりにくい表現 → 著者の意図を確認してから修正を提案
- 明らかな誤り → 修正する
- 好みの問題 → 原文を尊重する
```
